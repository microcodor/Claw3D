### 连接流程

```
1️⃣  浏览器发起连接
    ↓
    ws://192.168.0.132:3000/api/gateway/ws
    ↓
2️⃣  Claw3D 接收连接
    ↓
    server/index.js → handleUpgrade()
    ↓
3️⃣  WebSocket 代理处理
    ↓
    server/gateway-proxy.js → createGatewayProxy()
    ↓
4️⃣  代理连接到 OpenClaw
    ↓
    ws://localhost:18789
    ↓
5️⃣  OpenClaw Gateway 接收连接
    ↓
6️⃣  双向消息转发
    ↓
    浏览器 ←→ Claw3D 代理 ←→ OpenClaw Gateway
```

### 配置要点

#### Claw3D (.env)
```bash
# 允许局域网访问
HOST=0.0.0.0
PORT=3000

# 服务器端连接本地 OpenClaw
CLAW3D_GATEWAY_URL=ws://localhost:18789
CLAW3D_GATEWAY_TOKEN=your_token_here
CLAW3D_GATEWAY_ADAPTER_TYPE=openclaw

# 允许的上游主机
UPSTREAM_ALLOWLIST=localhost:18789,127.0.0.1:18789

# 访问令牌（内网可选）
# STUDIO_ACCESS_TOKEN=
```

#### OpenClaw
```bash
# 只监听本地（不暴露到局域网）
gateway.host = 127.0.0.1
gateway.port = 18789
gateway.auth.token = "强密码"
```

### 配置要点

#### Claw3D (.env)
```bash
HOST=0.0.0.0
PORT=3000

# 跨机器连接
CLAW3D_GATEWAY_URL=ws://192.168.0.133:18789
CLAW3D_GATEWAY_TOKEN=your_token_here

# 允许跨机器连接
UPSTREAM_ALLOWLIST=192.168.0.133:18789

# 必须设置访问令牌
STUDIO_ACCESS_TOKEN=强密码
```

#### OpenClaw
```bash
# 监听所有接口（暴露到局域网）
gateway.host = 0.0.0.0
gateway.port = 18789
gateway.auth.token = "强密码"
```

### ⚠️ 不推荐原因

1. **安全性降低**: OpenClaw Gateway 直接暴露到局域网
2. **网络延迟**: 跨机器通信增加延迟
3. **配置复杂**: 需要配置防火墙、网络策略
4. **维护困难**: 两台服务器需要分别维护

## 📊 架构对比

| 特性 | 同机部署 | 分机部署 |
|------|---------|---------|
| **安全性** | ⭐⭐⭐⭐⭐ OpenClaw 不暴露 | ⭐⭐⭐ OpenClaw 暴露到局域网 |
| **延迟** | ⭐⭐⭐⭐⭐ <1ms (localhost) | ⭐⭐⭐ 1-5ms (局域网) |
| **配置复杂度** | ⭐⭐⭐⭐⭐ 简单 | ⭐⭐⭐ 复杂 |
| **维护成本** | ⭐⭐⭐⭐⭐ 低 | ⭐⭐⭐ 高 |
| **资源利用** | ⭐⭐⭐ 单机资源 | ⭐⭐⭐⭐⭐ 分布式资源 |
| **推荐度** | ✅ **强烈推荐** | ⚠️ 特殊场景 |

## 🎯 推荐部署方案

### 小型部署（1-10 用户）

```
单台服务器
├── Claw3D Studio (0.0.0.0:3000)
└── OpenClaw Gateway (127.0.0.1:18789)

配置：
- 4 核 CPU
- 8GB 内存
- 100GB 存储
```

### 中型部署（10-50 用户）

```
单台服务器（配置升级）
├── Claw3D Studio (0.0.0.0:3000)
└── OpenClaw Gateway (127.0.0.1:18789)

配置：
- 8 核 CPU
- 16GB 内存
- 500GB 存储
```

### 大型部署（50+ 用户）

```
方案 1: 单机高配
├── Claw3D Studio (0.0.0.0:3000)
└── OpenClaw Gateway (127.0.0.1:18789)

配置：
- 16 核 CPU
- 32GB 内存
- 1TB 存储

方案 2: 负载均衡（多个 Claw3D 实例）
├── Nginx 负载均衡器
├── Claw3D Studio 1 (0.0.0.0:3001)
├── Claw3D Studio 2 (0.0.0.0:3002)
└── OpenClaw Gateway (127.0.0.1:18789)
```

## 🔧 快速部署命令

### 同机部署（推荐）

```bash
# 1. 启动 OpenClaw Gateway
openclaw gateway start

# 2. 配置 Claw3D
cd Claw3D
cat > .env << EOF
HOST=0.0.0.0
PORT=3000
CLAW3D_GATEWAY_URL=ws://localhost:18789
CLAW3D_GATEWAY_TOKEN=$(openclaw config get gateway.auth.token)
CLAW3D_GATEWAY_ADAPTER_TYPE=openclaw
UPSTREAM_ALLOWLIST=localhost:18789,127.0.0.1:18789
EOF

# 3. 启动 Claw3D
npm run build
pm2 start ecosystem.config.js

# 4. 验证
./verify-lan-config.sh
```

### 访问地址

- **本机**: http://localhost:3000/office
- **局域网**: http://192.168.0.132:3000/office
