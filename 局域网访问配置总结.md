### 1. 代码修改

**文件**: `server/network-policy.js`

**修改内容**:
- 注释掉了 `assertPublicHostAllowed` 函数中的错误抛出
- 添加了安全警告信息
- 允许在没有 `STUDIO_ACCESS_TOKEN` 的情况下绑定到 `0.0.0.0`

```javascript
// 修改前：强制要求 token
throw new Error(
  `Refusing to bind Studio to public host "${normalized}" without STUDIO_ACCESS_TOKEN.`
);

// 修改后：仅警告，不阻止启动
console.warn(
  `⚠️  Studio is binding to public host "${normalizeHost(host) || host}" without STUDIO_ACCESS_TOKEN.`
);
console.warn("   This is only safe in trusted internal networks.");
```

```bash

# HOST=0.0.0.0

```

## 🔒 安全说明

### ⚠️ 重要安全提示

此配置**仅适用于受信任的内网环境**：

✅ **适用场景**:
- 公司内网环境
- 家庭局域网
- 开发测试环境
- 大屏展示系统（内网）

❌ **不适用场景**:
- 公网服务器
- 云服务器（阿里云、腾讯云等）
- 不受信任的网络环境
- 需要对外提供服务的场景

### 🛡️ 公网部署安全措施

如果需要公网部署，必须采取以下安全措施：

1. **设置访问令牌**:
   ```bash
   STUDIO_ACCESS_TOKEN=your_very_strong_password_here_min_32_chars
   ```

2. **配置 HTTPS**:
   - 使用 Let's Encrypt 免费证书
   - 或使用反向代理（Nginx/Caddy）配置 SSL

3. **配置防火墙**:
   ```bash
   # 仅允许特定 IP 访问
   sudo ufw allow from 1.2.3.4 to any port 3000
   ```

4. **使用反向代理**:
   - Nginx
   - Caddy
   - Traefik

5. **IP 白名单**:
   - 在 Nginx/Caddy 中配置 IP 白名单
   - 或使用云服务商的安全组规则

## 🔧 故障排查

### 问题 1: 局域网无法访问

**症状**: 其他设备无法访问 `http://192.168.0.132:3000`

**排查步骤**:

1. **检查服务是否运行**:
   ```bash
   pm2 status
   # 或
   lsof -i :3000
   ```

2. **检查 HOST 配置**:
   ```bash
   cat .env | grep HOST
   # 应该显示: HOST=0.0.0.0
   ```

3. **检查防火墙**:
   ```bash
   # macOS
   sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate
   
   # Linux
   sudo ufw status
   ```

4. **检查网络连通性**:
   ```bash
   ping 192.168.0.132
   ```

5. **检查端口监听**:
   ```bash
   netstat -an | grep 3000
   # 应该显示: *.3000 或 0.0.0.0:3000
   ```

### 问题 2: 启动时报错

**症状**: `Refusing to bind Studio to public host "0.0.0.0" without STUDIO_ACCESS_TOKEN`

**解决方案**:
确认 `server/network-policy.js` 已正确修改。检查 Git 状态：

```bash
git log --oneline -1
# 应该显示包含 "配置局域网访问支持" 的提交
```

如果没有，重新拉取代码：
```bash
git pull origin main
```

### 问题 3: 页面加载但显示连接错误

**症状**: 页面能打开，但显示 "无法连接到 Gateway"

**说明**: 这是正常的。Claw3D 需要连接到 OpenClaw Gateway。

**解决方案**:
- 如果只是展示 SCR-02 和 SCR-03，不需要 Gateway
- 如果需要完整功能，请启动 OpenClaw Gateway
