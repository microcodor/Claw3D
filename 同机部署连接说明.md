# Claw3D 与 OpenClaw 同机部署连接说明

## 📋 问题回答

**问题**: 如果将 Claw3D 和 OpenClaw 服务都部署到一台服务器上，然后通过局域网访问 Claw3D 服务，能够联通 OpenClaw 服务吗？

**答案**: ✅ **可以！完全没有问题。**

---

## 🔍 连接原理

### 架构说明

Claw3D 使用了 **WebSocket 代理模式**，连接流程如下：

```
┌─────────────────────────────────────────────────────────────────┐
│                        局域网客户端                              │
│                    (浏览器 192.168.0.x)                         │
└────────────────────────────┬────────────────────────────────────┘
                             │ HTTP/WebSocket
                             │ ws://192.168.0.132:3000/api/gateway/ws
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                    服务器 (192.168.0.132)                       │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │              Claw3D Studio (Node.js)                     │  │
│  │              监听: 0.0.0.0:3000                          │  │
│  │                                                           │  │
│  │  ┌────────────────────────────────────────────────────┐ │  │
│  │  │  WebSocket Proxy (gateway-proxy.js)               │ │  │
│  │  │  - 接收来自浏览器的 WebSocket 连接                │ │  │
│  │  │  - 转发到本地 OpenClaw Gateway                    │ │  │
│  │  │  - 双向消息转发                                   │ │  │
│  │  └────────────────────────────────────────────────────┘ │  │
│  └──────────────────────┬───────────────────────────────────┘  │
│                         │ localhost                            │
│                         │ ws://localhost:18789                 │
│                         ▼                                       │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │         OpenClaw Gateway                                 │  │
│  │         监听: 127.0.0.1:18789                            │  │
│  └──────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 关键点

1. **浏览器 → Claw3D**: 通过局域网 IP 连接（`ws://192.168.0.132:3000/api/gateway/ws`）
2. **Claw3D → OpenClaw**: 通过 localhost 连接（`ws://localhost:18789`）
3. **代理转发**: Claw3D 的 Node.js 服务器充当 WebSocket 代理，在浏览器和 OpenClaw 之间转发消息

---

## ✅ 为什么可以联通

### 1. WebSocket 代理机制

Claw3D 的 `server/gateway-proxy.js` 实现了完整的 WebSocket 代理：

- **接收**: 接收来自浏览器的 WebSocket 连接
- **转发**: 建立到 OpenClaw Gateway 的 WebSocket 连接
- **双向通信**: 在两个连接之间双向转发所有消息

### 2. 服务器端连接

关键配置在 `.env` 文件中：

```bash
# 浏览器端配置（构建时）
NEXT_PUBLIC_GATEWAY_URL=ws://localhost:18789

# 服务器端配置（运行时，优先级更高）
CLAW3D_GATEWAY_URL=ws://localhost:18789
CLAW3D_GATEWAY_TOKEN=your_token_here
CLAW3D_GATEWAY_ADAPTER_TYPE=openclaw
```

**重要**: `CLAW3D_GATEWAY_URL` 使用 `localhost`，因为：
- Claw3D 和 OpenClaw 在同一台服务器上
- 服务器端代码通过 localhost 连接本地的 OpenClaw Gateway
- 这是服务器到服务器的本地连接，不经过网络

### 3. 浏览器端连接

浏览器通过局域网 IP 连接到 Claw3D：

```javascript
// 浏览器实际连接的地址
ws://192.168.0.132:3000/api/gateway/ws
```

浏览器**不需要**直接连接 OpenClaw Gateway，所有通信都通过 Claw3D 代理。

## 🔧 配置步骤

### 1. OpenClaw 配置

确保 OpenClaw Gateway 监听在 localhost：

```bash
# 查看 OpenClaw Gateway 配置
openclaw config get gateway.host
# 应该是: 127.0.0.1 或 localhost

openclaw config get gateway.port
# 应该是: 18789

# 获取认证 token
openclaw config get gateway.auth.token
```

### 2. Claw3D 配置

编辑 `.env` 文件：

```bash
# 服务器配置 - 允许局域网访问
HOST=0.0.0.0
PORT=3000

# Gateway 配置 - 服务器端使用 localhost
CLAW3D_GATEWAY_URL=ws://localhost:18789
CLAW3D_GATEWAY_TOKEN=从上面获取的token
CLAW3D_GATEWAY_ADAPTER_TYPE=openclaw

# 允许连接的上游主机
UPSTREAM_ALLOWLIST=localhost:18789,127.0.0.1:18789

# 访问令牌（内网可选）
# STUDIO_ACCESS_TOKEN=
```

### 3. 启动服务

```bash
# 1. 启动 OpenClaw Gateway
openclaw gateway start

# 2. 启动 Claw3D
cd Claw3D
npm run build
pm2 start ecosystem.config.js

# 或开发模式
npm run dev
```

### 4. 访问测试

**本机访问**:
```
http://localhost:3000/office
```

**局域网访问**:
```
http://192.168.0.132:3000/office
```

## 🔍 连接流程详解

### 完整的消息流

```
1. 浏览器发起连接
   ↓
   ws://192.168.0.132:3000/api/gateway/ws
   ↓
2. Claw3D Node.js 服务器接收连接
   ↓
   server/index.js → handleUpgrade()
   ↓
3. WebSocket 代理处理
   ↓
   server/gateway-proxy.js → createGatewayProxy()
   ↓
4. 代理连接到 OpenClaw
   ↓
   ws://localhost:18789
   ↓
5. OpenClaw Gateway 接收连接
   ↓
6. 双向消息转发
   ↓
   浏览器 ←→ Claw3D 代理 ←→ OpenClaw Gateway
```

### 认证流程

```javascript
// 1. 浏览器发送 connect 请求
{
  type: "req",
  method: "connect",
  id: "request-id",
  params: {
    client: { id: "browser-client", mode: "web" }
  }
}

// 2. Claw3D 代理注入 token（如果浏览器未提供）
{
  type: "req",
  method: "connect",
  id: "request-id",
  params: {
    client: { id: "browser-client", mode: "web" },
    auth: { token: "从 CLAW3D_GATEWAY_TOKEN 获取" }
  }
}

// 3. 转发到 OpenClaw Gateway
// 4. OpenClaw 验证 token 并返回响应
// 5. 代理转发响应给浏览器
```

## 🛡️ 安全考虑

### 1. OpenClaw Gateway 安全

OpenClaw Gateway 只监听 localhost，不暴露到局域网：

```bash
# OpenClaw Gateway 配置
gateway.host = 127.0.0.1  # 只监听本地
gateway.port = 18789
gateway.auth.token = "强密码"
```

**优点**:
- ✅ OpenClaw 不直接暴露到局域网
- ✅ 所有外部访问必须通过 Claw3D 代理
- ✅ Claw3D 可以实施额外的访问控制

### 2. Claw3D 安全

```bash
# 内网环境（当前配置）
HOST=0.0.0.0
# STUDIO_ACCESS_TOKEN=  # 未设置

# 公网环境（推荐配置）
HOST=0.0.0.0
STUDIO_ACCESS_TOKEN=强密码  # 必须设置
```

### 3. 防火墙配置

```bash
# 服务器防火墙规则
# 允许局域网访问 Claw3D
sudo ufw allow from 192.168.0.0/24 to any port 3000

# OpenClaw Gateway 不需要开放端口（只监听 localhost）
```

## ❌ 常见问题

### 问题 1: 浏览器无法连接到 Gateway

**症状**: 页面显示 "无法连接到 Gateway"

**排查步骤**:

1. **检查 OpenClaw Gateway 是否运行**:
   ```bash
   openclaw gateway status
   lsof -i :18789
   ```

2. **检查 Claw3D 配置**:
   ```bash
   cat .env | grep GATEWAY
   # 确认 CLAW3D_GATEWAY_URL=ws://localhost:18789
   ```

3. **检查 token 是否正确**:
   ```bash
   openclaw config get gateway.auth.token
   # 对比 .env 中的 CLAW3D_GATEWAY_TOKEN
   ```

4. **查看 Claw3D 日志**:
   ```bash
   pm2 logs claw3d-studio --lines 50
   ```

### 问题 2: 连接被拒绝 "gateway_url_blocked"

**症状**: 错误信息 "Upstream gateway URL is not in the allowed hosts list"

**解决方案**:

检查 `UPSTREAM_ALLOWLIST` 配置：

```bash
# .env 文件
UPSTREAM_ALLOWLIST=localhost:18789,127.0.0.1:18789
```

### 问题 3: Token 认证失败

**症状**: 错误信息 "Authentication failed" 或 "Invalid token"

**解决方案**:

1. 重新获取 OpenClaw token:
   ```bash
   openclaw config get gateway.auth.token
   ```

2. 更新 `.env` 文件:
   ```bash
   CLAW3D_GATEWAY_TOKEN=新的token
   ```

3. 重启 Claw3D:
   ```bash
   pm2 restart claw3d-studio
   ```

### 问题 4: 局域网设备无法访问

**症状**: 局域网内其他设备无法打开 Claw3D 页面

**解决方案**:

参考 [局域网访问配置说明.md](./局域网访问配置说明.md)

## 📊 性能考虑

### 连接延迟

```
浏览器 → Claw3D: ~1-5ms (局域网)
Claw3D → OpenClaw: <1ms (localhost)
总延迟: ~2-6ms
```

### 并发连接

- Claw3D 代理支持多个并发浏览器连接
- 每个浏览器连接对应一个到 OpenClaw 的连接
- 建议最大并发连接数: 50-100（取决于服务器性能）

### 资源占用

```
Claw3D Node.js 进程:
- 内存: ~200-500MB
- CPU: 1-5%（空闲时）

OpenClaw Gateway:
- 内存: ~500MB-2GB（取决于 agent 数量）
- CPU: 5-20%（取决于任务负载）
```
