# Claw3D 局域网访问配置说明

## 配置步骤

### 1. 环境变量配置（.env 文件）

确保以下配置项正确设置：

```bash
# 绑定到所有网络接口，允许局域网访问
HOST=0.0.0.0
PORT=3000

# 不设置 STUDIO_ACCESS_TOKEN（内网环境）
# STUDIO_ACCESS_TOKEN=

# Gateway 配置（服务器端使用 localhost）
CLAW3D_GATEWAY_URL=ws://localhost:18789
CLAW3D_GATEWAY_TOKEN=你的token
CLAW3D_GATEWAY_ADAPTER_TYPE=openclaw
```

### 2. 代码修改

已修改 `server/network-policy.js` 文件，允许在没有 `STUDIO_ACCESS_TOKEN` 的情况下绑定到公共主机（0.0.0.0）。

**修改内容：**
- 注释掉了强制要求 token 的错误抛出
- 添加了警告信息，提醒仅在受信任的内网环境使用

### 3. 启动服务

```bash
# 开发模式
npm run dev

# 生产模式（使用 PM2）
pm2 start ecosystem.config.js
```

### 4. 访问方式

**服务器本机访问：**
```
http://localhost:3000
```

**局域网内其他设备访问：**
```
http://192.168.0.132:3000
```
（将 IP 地址替换为服务器的实际局域网 IP）

## 安全说明

⚠️ **重要安全提示：**

1. **仅限内网使用**：此配置移除了访问令牌保护，仅适用于受信任的内网环境
2. **不要暴露到公网**：如果需要公网访问，必须设置 `STUDIO_ACCESS_TOKEN`
3. **防火墙配置**：建议配置防火墙规则，仅允许特定 IP 段访问

## 恢复访问令牌保护

如果需要恢复访问令牌保护：

1. 在 `.env` 文件中设置：
   ```bash
   STUDIO_ACCESS_TOKEN=你的强密码
   ```

2. 重启服务：
   ```bash
   pm2 restart claw3d-studio
   ```

3. 访问时需要在浏览器中设置 cookie：
   - Cookie 名称：`studio_access`
   - Cookie 值：你设置的密码

## 网络拓扑

```
局域网设备 (192.168.0.x)
    ↓
Claw3D Studio (192.168.0.132:3000)
    ↓ (localhost)
OpenClaw Gateway (localhost:18789)
```

## 常见问题

### Q: 局域网内无法访问？
A: 检查以下项：
- 服务器防火墙是否开放 3000 端口
- HOST 是否设置为 0.0.0.0
- 服务是否正常运行（`pm2 status`）

### Q: 启动时报错 "Refusing to bind Studio to public host"？
A: 确认 `server/network-policy.js` 已正确修改，错误抛出已被注释

### Q: 如何查看服务器 IP？
A: 
```bash
# macOS/Linux
ifconfig | grep "inet "
# 或
ip addr show
```

## 配置文件位置

- 环境变量：`Claw3D/.env`
- 网络策略：`Claw3D/server/network-policy.js`
- PM2 配置：`Claw3D/ecosystem.config.js`
