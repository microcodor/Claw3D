## ✅ 修复的问题

### 1. ERR_SSL_PROTOCOL_ERROR 错误

**问题**: 局域网访问 `http://192.168.0.132:3000/office` 报错 `net::ERR_SSL_PROTOCOL_ERROR`

**原因**: 
- Next.js 配置中包含 HSTS (Strict-Transport-Security) 头
- CSP 配置中包含 `upgrade-insecure-requests` 指令
- 这些配置强制浏览器将 HTTP 升级为 HTTPS

**修复**:
- ✅ 移除 HSTS 头（注释掉 Strict-Transport-Security）
- ✅ 移除 CSP 中的 `upgrade-insecure-requests` 指令
- ✅ 允许内网 HTTP 访问

### 2. PM2 部署路径错误

**问题**: `ecosystem.config.js` 中的路径配置错误

**修复**:
```javascript
// 修改前
cwd: '/home/openclaw1/claw3d'

// 修改后
cwd: '/home/openclaw1/Claw3D'
```

### 3. HDR 环境文件加载问题

**问题**: 3D 环境使用外部 CDN 加载 HDR 文件，可能导致加载失败

**修复**:
- ✅ 下载 HDR 文件到本地 `public/hdri/potsdamer_platz_1k.hdr`
- ✅ 修改组件使用本地文件路径
- ✅ 提高加载速度和稳定性

## 🛡️ 安全说明

### HSTS 移除的影响

**移除原因**:
- 内网部署不需要 HTTPS
- HSTS 会强制浏览器使用 HTTPS
- 导致 HTTP 访问失败

**安全影响**:
- ✅ 内网环境：无影响（内网本身是受信任的）
- ⚠️ 公网环境：如果部署到公网，建议重新启用 HSTS

**如何重新启用 HSTS**（公网部署）:

编辑 `next.config.ts`:

```typescript
// 取消注释以下代码
if (process.env.NODE_ENV === "production") {
  securityHeaders.push({
    key: "Strict-Transport-Security",
    value: "max-age=31536000; includeSubDomains",
  });
}
```

**前提**: 必须配置 HTTPS 证书
