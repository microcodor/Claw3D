# 局域网访问快速配置指南

## 一键配置（3步完成）

### 1️⃣ 修改 .env 文件

```bash
# 编辑 .env 文件
HOST=0.0.0.0
PORT=3000

# 注释掉或删除 STUDIO_ACCESS_TOKEN
# STUDIO_ACCESS_TOKEN=
```

### 2️⃣ 启动服务

```bash
# 开发模式
npm run dev

# 或生产模式（PM2）
npm run build
pm2 start ecosystem.config.js
```

### 3️⃣ 访问应用

**本机访问**：
```
http://localhost:3000/office
```

**局域网其他设备访问**：
```
http://192.168.0.132:3000/office
```
（将 IP 替换为服务器实际 IP）

## 验证配置

启动服务后，应该看到以下警告信息（这是正常的）：

```
⚠️  Studio is binding to public host "0.0.0.0" without STUDIO_ACCESS_TOKEN.
   This is only safe in trusted internal networks.
```

这表示配置成功，服务已允许局域网访问。

## 防火墙配置（可选）

### macOS
```bash
# 允许 3000 端口
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/local/bin/node
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblockapp /usr/local/bin/node
```

### Linux (ufw)
```bash
sudo ufw allow 3000/tcp
sudo ufw reload
```

### CentOS/RHEL (firewalld)
```bash
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload
```

## 故障排查

### ❌ 局域网无法访问

**检查项**：
1. 确认 `HOST=0.0.0.0` 已设置
2. 确认服务正在运行：`pm2 status` 或 `lsof -i :3000`
3. 确认防火墙已开放 3000 端口
4. 确认设备在同一局域网内
5. 尝试 ping 服务器 IP：`ping 192.168.0.132`

### ❌ 启动时报错 "Refusing to bind Studio to public host"

**解决方案**：
确认 `server/network-policy.js` 已正确修改（错误抛出已注释）

### ❌ 页面加载但无法连接 Gateway

**说明**：
这是正常的。Claw3D 需要连接到 OpenClaw Gateway 才能显示完整功能。如果只是展示 SCR-02 和 SCR-03 的模拟数据，不需要 Gateway。

## 安全建议

⚠️ **重要**：此配置仅适用于受信任的内网环境

- ✅ 适用：公司内网、家庭网络、开发环境
- ❌ 不适用：公网服务器、云服务器、不受信任的网络

如需公网部署，请：
1. 设置强密码：`STUDIO_ACCESS_TOKEN=your_strong_password_here`
2. 配置 HTTPS
3. 使用反向代理（Nginx/Caddy）
4. 配置 IP 白名单
