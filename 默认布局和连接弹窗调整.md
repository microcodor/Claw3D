# 默认布局和连接弹窗调整

## 修改内容

### 1. 默认布局调整
**修改前**：首次进入默认是单屏模式，显示 SCR-01  
**修改后**：首次进入默认是三横屏展示模式，同时显示 SCR-01、SCR-02、SCR-03

### 2. 连接弹窗调整
**修改前**：
- 首次进入自动尝试连接 OpenClaw Gateway
- 连接失败后自动弹出配置弹窗
- 用户被动看到连接错误

**修改后**：
- 首次进入不自动弹出配置弹窗
- 即使连接 OpenClaw WS 失败也不展示弹窗
- 用户需要手动点击右上角"连接"按钮才展示配置弹窗
- 更加简洁，不打扰用户

## 修改的文件

### 1. useOfficeLayout.ts
修改默认状态：

```typescript
// 修改前
screenMode: 'single',  // 单屏模式

// 修改后
screenMode: 'multi',   // 多屏模式（三横屏）
```

**说明**：
- `screenMode: 'multi'` - 多屏模式
- `layoutDirection: 'row'` - 横向排列（已经是默认值）
- `activeScreen: 'office'` - 默认激活 SCR-01（保持不变）

### 2. OfficeScreen.tsx
修改连接弹窗显示逻辑：

```typescript
// 修改前
const showGatewayLoadingOverlay =
  !agentsLoaded &&
  (!connectPromptReady ||
    (gatewayUrl.trim().length > 0 &&
      !shouldPromptForConnect &&
      ((!didAttemptGatewayConnect && showDelayedGatewayLoadingOverlay) ||
        (status === "connecting" && showDelayedGatewayLoadingOverlay))));

const showGatewayConnectOverlay =
  manualConnectDialogOpen ||
  (connectPromptReady &&
    status === "disconnected" &&
    !agentsLoaded &&
    didAttemptGatewayConnect &&
    (shouldPromptForConnect || showDelayedGatewayConnectOverlay));

// 修改后
const showGatewayLoadingOverlay = false; // 禁用自动加载覆盖层

const showGatewayConnectOverlay = manualConnectDialogOpen; // 只在手动点击时显示
```

**说明**：
- 完全禁用自动弹出的加载覆盖层
- 连接弹窗只在 `manualConnectDialogOpen` 为 true 时显示
- `manualConnectDialogOpen` 只在用户点击右上角"连接"按钮时设为 true

## 用户体验流程

### 首次访问
```
1. 用户访问 http://localhost:3000/office
   ↓
2. 页面加载，显示三横屏布局
   ├── 左侧：SCR-01 (网络安全学堂·AI实验室)
   ├── 中间：SCR-02 (热搜舆情中心 V2)
   └── 右侧：SCR-03 (创作工作室 V2)
   ↓
3. 右上角显示连接状态：DISCONNECTED (红点)
   ↓
4. 用户可以：
   ├── 直接使用 SCR-02 和 SCR-03（使用模拟数据）
   ├── 点击屏幕选择器切换单屏模式
   ├── 点击布局切换按钮改变排列方向
   └── 点击"连接"按钮手动配置 Gateway 连接
```

### 手动连接流程
```
1. 用户点击右上角"连接"按钮
   ↓
2. 弹出 Gateway 连接配置弹窗
   ├── 输入 Gateway URL
   ├── 输入 Token（可选）
   └── 点击"连接"
   ↓
3. 连接成功：
   ├── 状态变为 CONNECTED (绿点)
   ├── SCR-01 可以使用真实 Agent 数据
   └── 弹窗自动关闭
   ↓
4. 连接失败：
   ├── 状态保持 DISCONNECTED (红点)
   ├── 弹窗显示错误信息
   └── 用户可以重试或关闭弹窗
```

## 清除旧配置

如果之前访问过页面，localStorage 中可能保存了旧的配置。要使用新的默认配置，可以：

### 方法1：清除 localStorage
在浏览器控制台执行：
```javascript
localStorage.removeItem('office-layout-storage');
location.reload();
```

### 方法2：硬刷新
- Mac: `Cmd + Shift + R`
- Windows: `Ctrl + Shift + R`

### 方法3：清除浏览器数据
- 打开开发者工具（F12）
- Application → Storage → Clear site data

## 注意事项

1. **localStorage 持久化**
   - 用户的布局选择会保存到 localStorage
   - 下次访问时会恢复上次的配置
   - 首次访问或清除配置后使用默认值

2. **连接状态**
   - 连接状态不影响 SCR-02 和 SCR-03 的使用
   - 这两个屏幕使用模拟数据，无需 Gateway 连接
   - 只有 SCR-01 需要 Gateway 连接才能使用真实 Agent

3. **性能优化**
   - SCR-02 和 SCR-03 使用 lazy loading
   - 首次加载时会显示 loading 状态
   - 加载完成后正常显示
