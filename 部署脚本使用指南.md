## 🎯 快速开始（推荐流程）

### 步骤 1: 登录服务器

```bash
ssh openclaw1@192.168.0.132
```

### 步骤 2: 下载并运行部署脚本

**方式 A: 直接创建脚本文件**

```bash
# 创建脚本文件
cat > ~/deploy.sh << 'EOF'
#!/bin/bash
set -e
cd ~

echo "🚀 开始部署 Claw3D..."

if [ -d "claw3d" ]; then
    echo "📦 更新项目..."
    cd claw3d
    [ -f ".env" ] && cp .env .env.backup
    git pull || true
    [ -f ".env.backup" ] && mv .env.backup .env
else
    echo "📦 克隆项目..."
    git clone https://github.com/microcodor/Claw3D.git claw3d
    cd claw3d
fi

if [ ! -f ".env" ]; then
    echo "⚙️  配置环境变量..."
    cp .env.example .env
    
    if command -v openclaw &> /dev/null; then
        TOKEN=$(openclaw config get gateway.auth.token 2>/dev/null || echo "")
        [ -n "$TOKEN" ] && sed -i "s|^CLAW3D_GATEWAY_TOKEN=.*|CLAW3D_GATEWAY_TOKEN=${TOKEN}|g" .env
    fi
    
    sed -i "s|^# *HOST=.*|HOST=0.0.0.0|g" .env
    sed -i "s|^# *PORT=.*|PORT=3000|g" .env
    sed -i "s|^# *CLAW3D_GATEWAY_URL=.*|CLAW3D_GATEWAY_URL=ws://localhost:18789|g" .env
    sed -i "s|^# *CLAW3D_GATEWAY_ADAPTER_TYPE=.*|CLAW3D_GATEWAY_ADAPTER_TYPE=openclaw|g" .env
    sed -i "s|^# *STUDIO_ACCESS_TOKEN=.*|STUDIO_ACCESS_TOKEN=ChangeMe$(date +%s)|g" .env
    
    echo "⚠️  请修改 .env 中的 STUDIO_ACCESS_TOKEN 为强密码！"
fi

echo "📥 安装依赖..."
npm install

echo "🔨 构建项目..."
npm run build

echo "🚀 启动服务..."
pm2 stop claw3d 2>/dev/null || true
pm2 delete claw3d 2>/dev/null || true
pm2 start npm --name "claw3d" -- start
pm2 save

echo ""
echo "✅ 部署完成！"
echo "访问地址: http://$(hostname -I | awk '{print $1}'):3000"
echo "查看日志: pm2 logs claw3d"
echo "批准设备: openclaw devices approve --latest"
EOF

# 运行脚本
bash ~/deploy.sh
```

**方式 B: 如果项目已经在本地**

将脚本文件上传到服务器后运行：

```bash
# 在服务器上
bash server-deploy.sh
```

### 步骤 3: 修改访问令牌（重要！）

```bash
cd ~/claw3d
nano .env
```

找到并修改：
```bash
STUDIO_ACCESS_TOKEN=YourSecurePassword123!
```

保存：`Ctrl+O` → `Enter` → `Ctrl+X`

### 步骤 4: 重启服务

```bash
pm2 restart claw3d
```

### 步骤 5: 配置防火墙

```bash
# Ubuntu/Debian
sudo ufw allow 3000/tcp

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload
```

### 步骤 6: 访问并批准设备

1. 在浏览器访问：`http://192.168.0.132:3000`
2. 输入 STUDIO_ACCESS_TOKEN
3. 连接到 Gateway
4. 在服务器上批准设备：

```bash
openclaw devices approve --latest
```

## 🐛 常见问题

### Q1: 脚本执行失败怎么办？
**A**: 查看错误信息，通常是环境问题：
```bash
# 检查 Node.js
node -v  # 需要 v20+

# 检查 npm
npm -v   # 需要 v10+

# 检查 PM2
pm2 -v

# 检查 OpenClaw
openclaw gateway status
```

### Q2: 无法从局域网访问？
**A**: 检查以下几点：
```bash
# 1. 检查服务是否监听 0.0.0.0
netstat -tulpn | grep 3000

# 2. 检查防火墙
sudo ufw status

# 3. 检查 .env 配置
cat ~/claw3d/.env | grep HOST
```

### Q3: Gateway 连接失败？
**A**: 
```bash
# 1. 检查 Gateway 状态
openclaw gateway status

# 2. 重启 Gateway
openclaw gateway restart

# 3. 验证 Token
openclaw config get gateway.auth.token

# 4. 检查 .env 中的 token 是否正确
cat ~/claw3d/.env | grep CLAW3D_GATEWAY_TOKEN
```

### Q4: 构建失败？
**A**:
```bash
# 清理并重新构建
cd ~/claw3d
rm -rf node_modules .next
npm cache clean --force
npm install
npm run build
```
