# 无限循环错误修复说明

## 🐛 错误信息

```
Maximum update depth exceeded. This can happen when a component calls setState 
inside useEffect, but useEffect either doesn't have a dependency array, or one 
of the dependencies changes on every render.
```

## 🔍 问题原因

### 问题 1: GatewayClient.ts - shouldPromptForConnect 循环

在尝试修复 Gateway 弹窗闪屏问题时，我修改了 `shouldPromptForConnect` 的计算逻辑，导致无限循环。

**已修复** - 详见下文

### 问题 2: RetroOffice3D.tsx - Ref 作为依赖

**文件**: `src/features/retro-office/RetroOffice3D.tsx` (第 2944 行)

**问题代码**:
```typescript
useEffect(() => {
  const syncRenderAgentUi = () => {
    const next: Record<string, RenderAgentUiSnapshot> = {};
    for (const agent of renderAgentsRef.current) {
      next[agent.id] = {
        state: agent.state,
        status: agent.status,
      };
    }
    setRenderAgentUiById(next);  // 触发重渲染
  };

  syncRenderAgentUi();
  const timer = window.setInterval(syncRenderAgentUi, 250);
  return () => {
    window.clearInterval(timer);
  };
}, [renderAgentsRef]);  // ❌ 问题：Ref 作为依赖
```

**为什么会循环**:
1. `renderAgentsRef` 是一个 ref，它的引用永远不会改变
2. 但 React 会在每次渲染时检查依赖
3. 虽然引用相同，但可能触发不必要的重新执行
4. `setRenderAgentUiById` 触发重渲染
5. 形成无限循环

### 问题 3: useTaskBoardController.ts - 数组引用依赖

**文件**: `src/features/office/tasks/useTaskBoardController.ts` (第 964-977 行)

**问题代码**:
```typescript
useEffect(() => {
  if (!hydratedRef.current) return;
  const nextCards = stateRef.current.cards.map((card) =>
    syncCardWithAgent(syncCardWithLinkedRun(card, runLog), agents),
  );
  const changed = nextCards.some(
    (card, index) => card !== stateRef.current.cards[index],
  );
  if (!changed) return;
  dispatch({
    type: "hydrate",
    preference: { ...stateRef.current, cards: sortTaskBoardCards(nextCards) },
  });
}, [agents, runLog]);  // ❌ 问题：数组引用作为依赖
```

**为什么会循环**:
1. `agents` 和 `runLog` 是数组，每次渲染可能获得新的引用
2. 即使内容相同，引用不同会触发 useEffect
3. useEffect 调用 dispatch 更新状态
4. 状态更新触发重渲染
5. 重渲染又产生新的数组引用
6. 形成无限循环

**用户症状**:
- 初次进入时屏幕闪烁 3 次
- Agent 动画延迟加载
- 控制台显示无限循环错误

**已修复** - 详见 `useTaskBoardController无限循环修复说明.md`

**正确的做法**:
- Ref 不应该作为 useEffect 的依赖
- 使用空依赖数组 `[]` 只在挂载时运行一次
- setInterval 会持续更新，无需重新创建

## 💡 经验教训

### 为什么会导致无限循环？

1. **计算值作为依赖**：
   - `shouldPromptForConnect` 是一个计算值
   - 它被用作某些 useEffect 的依赖
   - 修改它的计算逻辑可能导致循环

2. **状态更新链**：
   ```
   shouldPromptForConnect 变化
     ↓
   useEffect 触发
     ↓
   setState 调用
     ↓
   组件重渲染
     ↓
   shouldPromptForConnect 重新计算
     ↓
   值又变化了
     ↓
   无限循环！
   ```

3. **Ref vs State**：
   - **Ref**：更新不会触发重渲染（安全）
   - **State**：更新会触发重渲染（可能导致循环）

### 如何避免？

1. **使用 useMemo**：
   ```typescript
   const shouldPromptForConnect = useMemo(() => {
     return settingsLoaded && status !== "connected" && ...;
   }, [settingsLoaded, status, ...]);  // 明确依赖
   ```

2. **使用 Ref 存储标志**：
   ```typescript
   const suppressPromptRef = useRef(false);
   
   // 在连接成功后设置
   if (status === "connected") {
     suppressPromptRef.current = true;
   }
   ```

3. **添加防抖/节流**：
   ```typescript
   const [showPrompt, setShowPrompt] = useState(false);
   
   useEffect(() => {
     const timer = setTimeout(() => {
       setShowPrompt(shouldPromptForConnect);
     }, 100);  // 延迟更新
     return () => clearTimeout(timer);
   }, [shouldPromptForConnect]);
   ```

## 🔧 更安全的弹窗闪屏修复方案

如果要再次尝试修复弹窗闪屏问题，应该：

### 方案 A: 使用 Ref 标志

```typescript
const justConnectedRef = useRef(false);

// 在连接成功时设置
useEffect(() => {
  if (status === "connected") {
    justConnectedRef.current = true;
    setTimeout(() => {
      justConnectedRef.current = false;
    }, 2000);  // 2秒冷却期
  }
}, [status]);

// 在显示条件中使用
const showGatewayConnectOverlay =
  connectPromptReady &&
  status === "disconnected" &&
  !agentsLoaded &&
  !justConnectedRef.current &&  // 冷却期内不显示
  (shouldPromptForConnect || showDelayedGatewayConnectOverlay);
```

### 方案 B: 使用独立的状态

```typescript
const [suppressPrompt, setSuppressPrompt] = useState(false);

// 连接成功后抑制提示
useEffect(() => {
  if (status === "connected") {
    setSuppressPrompt(true);
    const timer = setTimeout(() => {
      setSuppressPrompt(false);
    }, 2000);
    return () => clearTimeout(timer);
  }
}, [status]);

// 在显示条件中使用
const showGatewayConnectOverlay =
  connectPromptReady &&
  status === "disconnected" &&
  !agentsLoaded &&
  !suppressPrompt &&
  (shouldPromptForConnect || showDelayedGatewayConnectOverlay);
```
