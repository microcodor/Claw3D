# Claw3D 无法访问 - 快速解决指南

## 🚨 立即执行（3个命令解决大部分问题）

在服务器上运行：

```bash
# 1. 运行诊断脚本
bash ~/claw3d/diagnose.sh

# 2. 运行修复脚本
bash ~/claw3d/fix-and-restart.sh

# 3. 查看日志
pm2 logs claw3d --lines 50
```

---

## 📋 常见问题和解决方案

### 问题 1: 服务未启动

**症状**: `pm2 list` 显示 claw3d 不存在或状态不是 online

**解决**:
```bash
cd ~/claw3d
pm2 stop claw3d 2>/dev/null || true
pm2 delete claw3d 2>/dev/null || true
npm install
npm run build
pm2 start npm --name "claw3d" -- start
pm2 save
pm2 logs claw3d
```

---

### 问题 2: 只能本地访问，局域网无法访问

**症状**: `curl http://localhost:3000` 成功，但 `http://192.168.0.132:3000` 失败

**原因**: 服务只监听 127.0.0.1，没有监听 0.0.0.0

**解决**:
```bash
cd ~/claw3d

# 1. 编辑 .env
nano .env

# 2. 确保这一行存在且正确（不要有 # 注释）
HOST=0.0.0.0

# 3. 保存并退出 (Ctrl+O, Enter, Ctrl+X)

# 4. 重启服务
pm2 restart claw3d

# 5. 验证监听地址
netstat -tulpn | grep 3000
# 应该显示 0.0.0.0:3000，而不是 127.0.0.1:3000
```

---

### 问题 3: 防火墙阻止

**症状**: 服务运行正常，但从其他设备无法访问

**解决**:

**Ubuntu/Debian**:
```bash
sudo ufw allow 3000/tcp
sudo ufw status
```

**CentOS/RHEL**:
```bash
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload
sudo firewall-cmd --list-ports
```

**验证**:
```bash
# 在服务器上测试
curl http://localhost:3000

# 从另一台设备测试
curl http://192.168.0.132:3000
```

---

### 问题 4: 端口被占用

**症状**: 启动失败，日志显示 "port already in use" 或 "EADDRINUSE"

**解决**:
```bash
# 查看占用端口的进程
netstat -tulpn | grep 3000
# 或
lsof -i :3000

# 杀死占用进程
kill -9 <PID>

# 重启 Claw3D
pm2 restart claw3d
```

---

### 问题 5: 构建失败

**症状**: `npm run build` 失败

**解决**:
```bash
cd ~/claw3d

# 清理所有缓存和构建文件
rm -rf node_modules .next dist build
npm cache clean --force

# 重新安装和构建
npm install
npm run build

# 重启服务
pm2 restart claw3d
```

---

### 问题 6: .env 配置错误

**症状**: 服务启动但无法连接 Gateway

**解决**:
```bash
cd ~/claw3d

# 1. 获取 Gateway Token
openclaw config get gateway.auth.token

# 2. 编辑 .env
nano .env

# 3. 确保以下配置正确:
HOST=0.0.0.0
PORT=3000
CLAW3D_GATEWAY_URL=ws://localhost:18789
CLAW3D_GATEWAY_TOKEN=<粘贴上面获取的 token>
CLAW3D_GATEWAY_ADAPTER_TYPE=openclaw
STUDIO_ACCESS_TOKEN=YourSecurePassword123!

# 4. 保存并重启
pm2 restart claw3d
```

---

### 问题 7: OpenClaw Gateway 未运行

**症状**: Claw3D 启动正常，但无法连接到 Gateway

**解决**:
```bash
# 检查 Gateway 状态
openclaw gateway status

# 如果未运行，启动 Gateway
openclaw gateway start

# 或重启
openclaw gateway restart

# 查看 Gateway 日志
openclaw gateway logs
```
